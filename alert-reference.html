Alerting architecture — June 2026 update
Before
  • Warning / High / Critical → Desktop tray app only
  • Critical + oncall: Platform → Tray app + PagerDuty after 10-min window
  • Silence in Grafana within window → PagerDuty suppressed
  • Repeat every 4 h until resolved
  • Maintenance: silence oncall = Platform in Grafana
  • Laravel Error Rate Spike → Error Diagnosis Webhook → Jira + Teams
After
  • All alerts → PagerDuty immediately (group_wait: 0s)
  • Warning / High → PagerDuty low urgency → Slack + tray app (no escalation)
  • Critical → PagerDuty high urgency → Slack + tray app + escalation policy
  • Tray app reads PagerDuty API — no Grafana dependency
  • Maintenance: PagerDuty maintenance windows
  • Laravel Error Rate Spike → Error Diagnosis Webhook only (unchanged)
Severity key
CRITICALPD  PagerDuty high urgency — Slack + escalation policy + tray app (purple, flash)
HIGHPD  PagerDuty low urgency — Slack + tray app (red, flash, no escalation)
WARNINGPD  PagerDuty low urgency — Slack + tray app (orange, no flash)
WEBHOOK  Laravel Error Rate Spike only — Error Diagnosis → Jira + Teams
